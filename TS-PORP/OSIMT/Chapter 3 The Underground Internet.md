# Chapter 3: The Underground Internet
## Internet layers
The public web contains pages that search engines can crawl, index, and return in ordinary search results. Search engines discover pages through links, sitemaps, and other signals, then store analysed information in large indexes. They do not index every page they discover. Pages can remain outside search results because they require a login, sit behind a search form, block crawlers, use low-quality or duplicate content, or exist in services that ordinary crawlers cannot reach.

The deep web covers web content that ordinary search engines do not expose in their public results. It includes database records returned after a user submits a form, authenticated portals, paywalled material, private cloud content, internal business systems, library catalogues, academic databases, and public records systems. The deep web is not inherently criminal. Most internet users encounter it whenever they use banking portals, subscription databases, email, or forms that retrieve records from a server.

The dark web is a smaller category of intentionally hidden services. It operates through darknet or anonymity networks that need special software or configuration. The terms deep web, dark web, and darknet should not be used as synonyms. The deep web describes unindexed web content. The dark web describes hidden online services. A darknet describes the network layer that supports those hidden services.
## Finding deep web material
Open source intelligence (OSINT) analysts often find deep web material by searching within the target site or using specialist databases, directories, public records systems, academic indexes, and Internet of Things search engines. This work requires judgement because each system indexes a different collection and applies different access rules. Good investigation records the search terms used, the date of access, the database searched, and any filtering applied.

Search engines for internet-connected devices can reveal exposed routers, webcams, servers, industrial systems, and other connected equipment. These tools can support defensive security by helping organisations identify misconfigured or vulnerable assets. They can also create risk when used carelessly, so investigators should use them within lawful authority and defined scope.
## Darknet users and risks
Darknets attract many lawful users. Journalists, human rights workers, whistleblowers, researchers, privacy advocates, intelligence services, law enforcement bodies, and companies investigating leaks may use anonymity networks to reduce surveillance and protect sources. At the same time, criminal actors use hidden services to advertise stolen data, drugs, forged documents, malware, exploit services, fraud material, and other unlawful goods or services.

Dark web activity is difficult to measure reliably. Hidden services change addresses, disappear quickly, restrict access, or require invitations. Samples can overrepresent criminal markets if researchers collect from public directories or known illicit forums. Analysts should treat broad claims about the size of the deep web, the number of onion services, or the percentage of criminal content as estimates rather than stable facts.

Cryptocurrency does not make darknet commerce fully anonymous. Bitcoin records transactions on a public blockchain and is better described as pseudonymous. Investigators can often link addresses to services, wallets, exchanges, or people through blockchain analysis, reuse patterns, exchange records, and operational errors. Some actors use privacy tools or privacy coins, but financial anonymity remains a matter of degree rather than a guarantee.
## Tor and onion services
Tor is the best-known anonymity network. Tor Browser routes traffic through a circuit of relays so that no single relay knows both the user's location and the destination. The entry relay can see the user's IP address but not the destination. The exit relay can see traffic leaving Tor to the public internet if the destination connection lacks HTTPS. Onion services work differently. They stay inside the Tor network, use .onion addresses, hide the service location, and provide end-to-end authentication and encryption between the client and the onion service.

Tor improves privacy but does not create perfect anonymity. A powerful adversary that can observe traffic entering and leaving the network may attempt timing correlation. Websites can still fingerprint behaviour, accounts can identify a user, malware can compromise the device, and documents can leak metadata. Tor may also be blocked or treated with suspicion in some jurisdictions, so legal and operational risk must be assessed before use.

Investigators using Tor should download Tor Browser from the official project, keep it updated, avoid extra browser extensions, separate identities by task, and prefer the highest security level where usability permits. Modern Tor Browser keeps JavaScript enabled at the standard security level, while the safest level disables JavaScript by default and restricts other risky browser features. Proxy gateways that expose onion services through ordinary browsers may help casual viewing, but they remove key anonymity benefits and should not be used for sensitive work.
## Tails for high-risk work
Tails is a security-focused operating system designed to run from external media and route network connections through Tor. It reduces forensic traces on the host computer because ordinary work disappears when the session shuts down, unless the user deliberately saves selected data in Persistent Storage. Persistent Storage uses encryption and a passphrase, but it is not hidden. An attacker who obtains the USB stick can see that persistent storage exists and may try to force, guess, or obtain the passphrase.

Tails is safer than an ordinary operating system for many anonymity tasks, but it is not magic. It cannot protect against compromised firmware, altered hardware, physical keyloggers, careless account use, unsafe downloads, or identity leakage through writing style and behaviour. Tails also does not automatically remove all metadata from files. It includes Metadata Cleaner, which can remove metadata from many common formats, but complex documents can still contain embedded material that carries its own metadata.

A disciplined OSINT workflow separates activities. Investigators should not use the same session, account, browser state, or document set for unrelated identities. High-risk work should use fresh sessions, clean files, current software, documented collection steps, and strict limits on downloading or opening unknown material.
## I2P, Freenet, and Hyphanet
I2P is a decentralised privacy network designed mainly for communication and services inside the network. It uses a distributed network database, packet switching, unidirectional tunnels, and garlic routing. These design choices support hidden services and peer-to-peer activity within I2P. Tor remains stronger for general anonymous access to the public web because it has a larger user base, more mature tooling, and many exit relays. I2P suits internal services and peer-to-peer applications better than ordinary web browsing.

Freenet changed substantially after the original project split in 2023. The original Freenet codebase continues as Hyphanet, a peer-to-peer system for privacy-respecting and censorship-resistant publishing. The newer Freenet project follows a different architecture. Older references to Freenet should therefore be checked carefully before relying on current documentation or tools.
## Search engines and OSINT workflow
General search engines organise public information through crawling, indexing, and ranking. Their results reflect crawler access, indexing decisions, ranking systems, localisation, language, device signals, query interpretation, and content quality. They do not provide a complete map of the internet.

Effective OSINT combines surface web search, deep web databases, specialist indexes, device search engines, and lawful darknet research where justified. Analysts should expect incomplete results and volatile sources. Strong work depends on search planning, source verification, repeatable collection notes, clear legal authority, and caution when accessing unknown services.

The underground internet is not a single hidden place. It is a set of access problems, network designs, privacy tools, databases, legal boundaries, and operational risks. Investigators who understand those distinctions can collect more accurately, protect their identities more effectively, and avoid confusing privacy infrastructure with criminal intent.
